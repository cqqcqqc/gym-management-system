# 🏋️‍♂️ 健身房管理系统（Gym Management System）

### 📝 项目简介
这是我独立开发的**健身房管理系统**，采用前后端分离架构，实现了会员管理、课程预约、数据统计等核心功能，是我全栈开发能力的综合实践。

### 🛠 技术栈
- **前端 (Frontend)**：微信小程序原生开发，负责用户交互与页面渲染。
- **后端 (Backend)**：Node.js + Express，提供 RESTful API 接口，处理业务逻辑与数据存储。
- **数据库**：本地 JSON 文件/轻量级数据库（可扩展至 MySQL）。

### 🚀 核心功能
- 👤 **会员管理**：实现会员注册、信息查询、到期提醒等功能。
- 📅 **课程预约**：支持用户在线预约健身课程，实时更新课程状态。
- 📊 **数据统计**：后台统计会员活跃度、课程参与率等关键数据，辅助运营决策。
- 📱 **移动端适配**：通过微信小程序，为用户提供便捷的移动端访问体验。

### 💡 项目亮点
- 采用前后端分离架构，代码结构清晰，便于维护和扩展。
- 后端服务运行于 `http://localhost:60624`，提供稳定的 API 支持。
- 独立完成从需求分析、技术选型到代码实现的全流程开发。

### 🧰 运行说明
1.  **启动后端服务**：
    ```bash
    cd server
    node server.js
    # 服务运行于 http://localhost:60624
    ```
2.  **启动前端小程序**：
    - 使用微信开发者工具打开 `gym-management-mini-program` 目录。
    - 编译并预览小程序。
